# Implicit-contract discovery family router

Use this before choosing one of the overlapping discovery records. The goal is
to keep generalization, tacit intent, latent-structure, closure, and
negative-space work complementary instead of duplicating the same reasoning in
multiple ledgers.

```text
task / artifact:
trigger:
visible-frame risk:
hidden-rule / implicit-contract risk:
unstated-user-intent risk:
transferable-rule / false-analogy risk:
redundancy / stale-surface risk:
chosen primary record:
supporting records, if needed:
why one record is enough:
what would require escalation to latent-structure ledger:
```

Route rules:

- Use `Generalization audit record` when the main risk is turning local facts
  into a transferable rule, detecting false analogies, or proving that a lucky
  success is not a general pattern.
- Use `Tacit intent recovery record` when the main risk is unstated user intent
  or implicit product/repo expectations in an underspecified request.
- Use the latent-structure ledger when the action is medium/high risk,
  externally visible, black-box, safety-relevant, or depends on an inferred
  hidden rule that needs a pre-action or final validator.
- Use `Closure check record` when the frame may be incomplete or the observed
  set must not be treated as exhaustive.
- Use `Negative-space discovery record` when the artifact appears complete but
  entailed companions, journey gaps, or private speculative neighbors need a
  bounded divergence pass.
- Use `Excess / redundancy / legacy-surface discovery record` when the question
  is whether to remove, deprecate, consolidate, or intentionally keep a surface.
